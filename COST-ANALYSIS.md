# Tone Generator 单次调用成本分析

## 核心结论

**单次用户调用的成本 ≈ $0.0000（几乎为零）**

Tone Generator 是纯前端工具，没有后端 API 调用，没有服务器成本，没有数据库查询。

---

## 成本拆解

### 1. 前端计算成本（用户浏览器承担）

| 操作 | 计算位置 | 成本 |
|------|---------|------|
| 生成正弦波 | Web Audio API（用户 CPU） | $0 |
| 生成方波/三角波/锯齿波 | Web Audio API（用户 CPU） | $0 |
| 频率调整 | JavaScript 计算（用户 CPU） | $0 |
| 波形可视化 | Canvas 渲染（用户 GPU/CPU） | $0 |
| 音量控制 | Web Audio API GainNode | $0 |
| 播放/暂停 | 浏览器原生 API | $0 |

**所有音频生成和信号处理都在用户浏览器本地完成，不消耗服务器资源。**

### 2. 静态资源成本（CDN/托管）

| 资源 | 大小 | Cloudflare Pages 费用 |
|------|------|----------------------|
| HTML 页面 | ~15 KB | $0（免费额度内） |
| JavaScript（Next.js 产物） | ~150 KB（gzip） | $0（免费额度内） |
| CSS | ~20 KB（gzip） | $0（免费额度内） |
| 字体/图标 | ~50 KB | $0（免费额度内） |
| **总计** | **~235 KB** | **$0** |

**Cloudflare Pages 免费额度：**
- 无限请求
- 无限带宽
- 500 次构建/月

### 3. 第三方服务成本

| 服务 | 用途 | 费用 |
|------|------|------|
| Google Analytics 4 | 流量分析 | $0（免费） |
| Plausible（可选） | 隐私友好分析 | ~$9/月（固定，不按调用） |
| 域名 | tonetool.net 等 | ~$10/年 |

### 4. 可能的扩展成本（P1 及以后）

| 功能 | 技术方案 | 成本 |
|------|---------|------|
| 噪声生成（白/粉/棕） | Web Audio API 本地生成 | $0 |
| 双耳节拍 | Web Audio API 立体声控制 | $0 |

## 对比：如果用了后端 API 的成本

假设我们做了一个"错误版本"，用后端生成音频：

| 方案 | 单次调用成本 | 月成本（10万次调用） |
|------|-------------|---------------------|
| **纯前端（正确方案）** | **$0** | **$0** |
| AWS Lambda + API Gateway 生成音频 | ~$0.0002 | ~$20 |
| 第三方音频 API（如某云服务） | ~$0.001-0.01 | ~$100-1000 |
| 自建服务器（ECS/VPS） | ~$0.0001 + 固定成本 | ~$10 + $20/月 |

**纯前端方案每月节省 $20-1000+**

## 唯一可能的"成本"

### 用户侧成本（不是我们的成本）
- **CPU 占用**：Web Audio API 计算占用用户设备 CPU，约 1-5%
- **电池消耗**：持续播放音频会增加手机耗电
- **带宽**：首次加载 ~235 KB，后续页面缓存

### 我们的实际支出
| 项目 | 月成本 | 年成本 |
|------|--------|--------|
| 域名 | ~$0.83 | ~$10 |
| Plausible 分析（可选） | ~$9 | ~$108 |
| **总计** | **~$9.83** | **~$118** |

## 为什么纯前端是 Tone Generator 的最佳架构

1. **Web Audio API 足够强大**
   - 原生支持振荡器、增益、滤波器、分析器
   - 延迟极低（< 10ms）
   - 无需服务器中转

2. **音频数据量小**
   - 不是传输音频文件，而是传输"生成参数"
   - 参数：频率（数字）、波形（字符串）、音量（数字）
   - 数据量：~50 字节 vs 音频文件 ~1 MB/分钟

3. **实时性要求高**
   - 用户拖动滑块时需要实时响应
   - 后端往返会有 50-200ms 延迟
   - 本地计算是零延迟

4. **可离线使用**
   - Service Worker 缓存后，无需网络也能使用
   - 竞品 szynalski.com 就是纯离线工具

## 成本优化建议

### 已经最优，无需优化
纯前端 + Cloudflare Pages 已经是成本最低架构。

### 如果未来要扩展（保持低成本）
| 扩展方向 | 低成本方案 | 避免 |
|---------|-----------|------|
| 用户保存预设 | LocalStorage（本地） | 后端数据库 |
| 分享设置 | URL 参数编码 | 后端存储 |
| 使用统计 | 匿名 GA4 事件 | 自建数据库 |
| Pro 版本验证 | Stripe 客户端验证 + 简单后端 | 复杂用户系统 |

## 总结

| 指标 | 数值 |
|------|------|
| 单次调用 API 成本 | **$0** |
| 单次用户会话成本 | **~$0.00001**（仅 CDN 带宽分摊） |
| 月运营成本（1万用户） | **~$10**（域名+可选分析） |
| 月运营成本（10万用户） | **~$10**（Cloudflare 免费无限带宽） |
| 月运营成本（100万用户） | **~$10-20**（仍几乎为零） |

**Tone Generator 是一个"零边际成本"产品——用户越多，单位成本越低，规模效应极强。**

# Tone Generator — 竞品视觉分析

## 竞品 1: onlinetonegenerator.com

- **主色调**: 白底 + 浅灰边框 + 蓝色点缀 (#0066CC)
- **深色/浅色**: 浅色主题
- **字体**: 系统默认 sans-serif (Arial/Helvetica 类)
- **Hero 策略**: 无 Hero。页面顶部直接是导航列表（20+个链接），然后一个巨大的广告 iframe，接着才是工具本身
- **布局**: 单栏居中，文字密集，无视觉层级
- **CTA 文案**: "Play" / "Stop" / "Save" — 纯功能按钮
- **整体感觉**: 2010 年代早期网站。 cluttered，广告密集，无品牌感，像维基百科+工具拼凑
- **可借鉴**: URL 分享功能、波形选择逻辑、频率输入框设计
- **必须避开**: 广告轰炸、无 Hero、导航列表过长、无移动端优化、无视觉设计

## 竞品 2: audiocheck.net

- **主色调**: 白底 + 黑色文字 + 蓝色链接
- **深色/浅色**: 浅色主题
- **字体**: Times New Roman / serif 正文 + sans-serif 标题（混合混乱）
- **Hero 策略**: 无 Hero。H1 "Welcome to Audio Check" 后直接是段落文字
- **布局**: 单栏，大量文字列表，右侧 donation 边栏
- **CTA 文案**: "Support us" 捐赠按钮
- **整体感觉**: 学术/个人博客风格。信息丰富但极其过时，像 2005 年的网页
- **可借鉴**: 内容深度（大量测试类型）、教育价值、听力安全警告
- **必须避开**: 字体混用、无现代布局、无品牌识别、donation 模式干扰体验

## 竞品 3: szynalski.com/tone-generator/

- **主色调**: 白底 + 黑色文字 + 红色点缀
- **深色/浅色**: 浅色主题
- **字体**: 系统默认 sans-serif
- **Hero 策略**: 无 Hero。H1 后直接是工具面板
- **布局**: 单栏，工具面板居中，下方是文字说明
- **CTA 文案**: "PLAY" / "SUPPORT THIS SITE »"
- **整体感觉**: 极简但简陋。工具优先，但无设计感，像原型而非产品
- **可借鉴**: 工具前置（不废话直接能用）、键盘快捷键提示、频率倍乘按钮
- **必须避开**: 过于简陋、无品牌、无移动端优化、广告嵌入

---

## 竞品共同弱点总结

| 弱点 | 我们的机会 |
|---|---|
| 全部浅色主题 | 我们优先深色主题，差异化明显 |
| 全部无 Hero / 无品牌感 | 我们做强品牌 Hero，工具嵌入首屏 |
| 全部广告密集或 donation 干扰 | 我们 Free 无广告，Pro 付费去广告 |
| 全部移动端体验差 | 我们 mobile-first 设计 |
| 全部视觉过时（2010 前风格） | 我们现代工业/赛博调性 |
| 全部无实时可视化 | 我们 Canvas 波形可视化 |
| 全部无定价体系 | 我们有 Free / Pro / Lifetime 三层 |

---

## 设计决策

基于竞品分析，Tone Generator 必须：

1. **深色主题** — 三个竞品全是浅色，深色是最大差异化
2. **工具嵌入 Hero** — 用户来这就是为了用工具，不要让他们滚动
3. **强品牌识别** — 竞品全部无品牌感，我们要像专业产品
4. **零广告干扰** — Free 版干净，Pro 版更干净
5. **移动端优先** — 竞品全部桌面思维
